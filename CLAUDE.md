# プロジェクトルール

## プロジェクト概要
本プロジェクトは、出版予定のPythonプログラミング入門書に掲載するアプリケーションを開発するものです。

## 関連ファイル
- 書籍原稿のドラフト: プロジェクト外の原稿ファイル（Git管理対象外）
  - この原稿ファイルは実際の書籍の執筆内容を含むため、Git管理対象外とする
  - 本プロジェクトのコードは、この原稿の内容に基づいて実装する

## 目的
- Raspberry Piを使用した複数サーボモーターの制御アプリケーションの開発
- Python初学者向けの教材として使用可能なコードの作成
- 入門書の読者が理解しやすい、クリーンで分かりやすいコード実装

## 開発方針
- コードは教育目的であることを考慮し、可読性を最優先する
- 適切なコメントを日本語で記述する
- 初学者でも理解できるシンプルな構造を維持する
- エラーハンドリングを適切に実装し、学習者が問題を理解しやすくする

### 重要な原則
1. **技術書掲載用サンプルコード**: 本プロジェクトのコードはプログラミング技術書に掲載するサンプルコードである。書籍の読者が実際に入力・実行することを前提とする。
2. **最小構成の実装**: 初心者にわかりやすいよう、必要最低限の機能に絞った最小構成で実装する。過度な抽象化や将来の拡張を見越した複雑な設計は避ける。

## 技術スタック
- Python 3
- Raspberry Pi GPIO制御
- サーボモーター制御ライブラリ

## コーディング規約
- PEP 8に準拠
- 関数・クラスには適切なdocstringを記述
- 変数名・関数名は分かりやすい日本語またはローマ字を使用可能
- マジックナンバーは定数として定義
- 詳細なコーディング規約は以下のドキュメントに従う：
  - `python_coding_guidelines.md`: Python全般のコーディング規約
  - `COMMENT_STYLE_GUIDE.md`: コメント記載の詳細ガイドライン

## セキュリティとプライバシー

### GitHub公開前の機密情報チェック
コードをGitHubにpush/公開する前に、必ず以下の項目をチェックすること：

#### チェック項目
- [ ] パスワード、APIキー、トークン等の認証情報が含まれていないか
- [ ] 秘密鍵（.pem, .key等）やクレデンシャルファイルが含まれていないか
- [ ] 個人を特定できるパス情報（ホームディレクトリのフルパス等）が含まれていないか
- [ ] プライベートなIPアドレスやホスト名が含まれていないか
- [ ] メールアドレス等の個人情報が含まれていないか
- [ ] .gitignoreが適切に設定されているか（.env, *.log, venv/等）

#### 機密情報が見つかった場合の対応
1. 該当箇所を削除または汎用的な表現に置き換える
2. 既にコミット済みの場合は、git historyから完全に削除する
3. 必要に応じて.gitignoreに追加する

#### 推奨ツール
- 手動チェック: `grep -rn "password\|secret\|api_key" .`
- 自動チェック: git-secrets, gitleaks等のツールの活用を推奨
